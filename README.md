# PSD Mobile Viewer

A modern, mobile-first Single Page Application (SPA) for viewing PSD and PSB files directly in the browser — no backend required. Inspired by [psdtool](https://oov.github.io/psdtool/).

[![Deploy to GitHub Pages](https://img.shields.io/badge/Deploy-GitHub%20Pages-blue?logo=github)](https://pages.github.com/)

---

## ✨ Features

- **100% client-side** — everything runs in the browser; no server needed
- **Drag & drop or file picker** — open `.psd` / `.psb` files instantly
- **Responsive canvas viewer** with smooth pinch-to-zoom, pan, and double-tap-to-fit
- **Mobile-optimised layer panel** — swipe-up bottom sheet with:
  - Visibility toggles per layer
  - Per-layer opacity sliders
  - Collapsible group/folder support
- **Real-time composite preview** with basic blend mode support (multiply, screen, overlay, …)
- **Text layer content** displayed when a text layer is selected
- **Export as PNG** — saves the current composite at full PSD resolution
- **"Load sample PSD"** — bundled demo file, no upload needed
- **PWA support** — installable, works offline after first visit
- **Dark theme** by default (designer-friendly)

## 🚀 Quick Start (local dev)

### Prerequisites

- [Node.js](https://nodejs.org/) v18 or later
- npm v9 or later

### 1. Install dependencies

```bash
npm install
```

### 2. Generate the bundled sample PSD (one-time)

```bash
node scripts/create-sample-psd.js
```

This writes `public/sample.psd` (~23 KB). You only need to run this once (or after deleting `public/sample.psd`).

### 3. Start the dev server

```bash
npm run dev
```

Open the URL printed in the terminal (usually <http://localhost:5173/>). The app hot-reloads on file changes.

## 🌐 Deploy to GitHub Pages

### Step 1 — Fork / push to GitHub

Make sure your code is in a GitHub repository. If you cloned this project, push it:

```bash
git remote add origin https://github.com/<YOUR_USER>/<YOUR_REPO>.git
git push -u origin main
```

### Step 2 — Set the correct `base` path

Open `vite.config.js` and update `base` to match **your repository name**:

```js
// vite.config.js
export default defineConfig({
  // Change '/psd-mobile-viewer/' to '/<YOUR_REPO_NAME>/'
  base: '/psd-mobile-viewer/',
  ...
});
```

> If you are deploying to a custom domain (e.g. `https://example.com`), set `base: '/'`.

### Step 3 — Build

```bash
npm run build
```

### Step 4 — Deploy the `dist/` folder

#### Option A — GitHub Actions (recommended)

Create `.github/workflows/deploy.yml`:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: true

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deploy.outputs.page_url }}
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - run: npm ci
      - run: node scripts/create-sample-psd.js
      - run: npm run build

      - uses: actions/configure-pages@v4
      - uses: actions/upload-pages-artifact@v3
        with:
          path: dist

      - id: deploy
        uses: actions/deploy-pages@v4
```

Then go to **Settings → Pages → Source** and select **GitHub Actions**.

#### Option B — Manual with `gh-pages`

```bash
npm install -D gh-pages
npx gh-pages -d dist
```

Then go to **Settings → Pages → Source** and select **Deploy from a branch → gh-pages**.

### Step 5 — Visit your app

Your app will be live at:

```
https://<YOUR_USER>.github.io/<YOUR_REPO>/
```

## 📁 Project Structure

```
psd-mobile-viewer/
├── index.html                 # App shell (HTML entry point)
├── vite.config.js             # Vite config — update base path here
├── tailwind.config.js         # Tailwind CSS config
├── postcss.config.js
├── package.json
├── public/
│   ├── .nojekyll              # Prevents GitHub Pages from running Jekyll
│   ├── manifest.json          # PWA web app manifest
│   ├── sw.js                  # Service worker (offline support)
│   ├── sample.psd             # Bundled demo PSD (generated by script)
│   └── icons/                 # PWA icons (generated by script)
├── scripts/
│   ├── create-sample-psd.js   # Generates public/sample.psd
│   └── create-icons.js        # Generates public/icons/*.png
└── src/
    ├── main.js                # App entry — wires everything together
    ├── style.css              # Tailwind + custom CSS
    ├── psd-renderer.js        # Canvas compositing with blend modes
    ├── layer-panel.js         # Bottom-sheet layer panel UI
    └── touch-handler.js       # Pinch-to-zoom, pan, double-tap gestures
```
